# Лекцийн сэдэв

**Лекц #5:** Өгөгдлийн сангийн удирдлагын бичиг баримт болон нууц үгийн бодлогыг хэрэгжүүлэх сэдвийн ойлголтыг бататтах даалгавар ажил

---

## Асуулт

Нууц үгийн бодлогод Account Lockout болон Password Expiration тохиргоо хийх нь ямар халдлагаас хамгаалдаг вэ?

---

## Хариулт

Нууц үгийн бодлогод **Account Lockout** болон **Password Expiration** тохиргоо хийх нь дараах халдлагуудаас хамгаалдаг:

- **Brute Force халдлага** — Нэвтрэх оролдлогыг тоогоор хязгаарлаж, давтан оролдохыг хаана
- **Dictionary Attack** — Үг хэллэгийн сан ашиглан нууц үгийг таахаас сэргийлнэ
- **Credential Stuffing** — Хулгайлагдсан нэвтрэх мэдээллийг ашиглах оролдлогыг блоклоно
- **Unauthorized Access** — Хуучин/задарсан нууц үгээр нэвтрэхийг Password Expiration-р хаана

- **Дүгнэлт:** Эдгээр бодлогууд нь системийн аюулгүй байдлыг нэмэгдүүлж, зөвшөөрөлгүй хандалтаас хамгаалдаг чухал хэрэгсэл юм.

---

## Жишээ

```
Account Lockout Policy:
- Оролдлогын тоо: 5
- Хаах хугацаа: 30 минут

Password Expiration Policy:
- Нууц үг солих хугацаа: 90 хоног
- Өмнөх нууц үгийг дахин ашиглах: Хориотой
```

---

## Дүгнэлт

Нууц үгийн бодлогыг зөв тохируулснаар:

-  Системийн аюулгүй байдал нэмэгдэнэ
-  Халдлагын эрсдэл буурна
-  Хэрэглэгчийн бүртгэл хамгаалагдана
-  Байгууллагын мэдээллийн аюулгүй байдал хангагдана
